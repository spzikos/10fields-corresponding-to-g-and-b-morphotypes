# Αντιστοίχιση 10 Πεδίων → Κωδικοί Υποδειγμάτων Β & Γ

> Πηγές: τα επίσημα Excel templates (`docs/ΥπόδειγμαΒ_ΕΛΠ.ZIP`, `docs/Υπόδειγμα_Γ_ΕΛΠ_ΠΜΕ.ZIP`),
> τα schemas (`pipeline/schemas/schemasG.py`, `schemasB.py`, `schema.py`) και το `docs/mds/Υπόδειγμα_Β_Κανόνες.md`.

**Κρίσιμη προειδοποίηση:** ο **ίδιος αριθμητικός κωδικός σημαίνει διαφορετικό πράγμα** στο Γ και στο Β.
Το **Γ** (ΕΛΠ ΠΜΕ — Πολύ Μικρές Οντότητες) χρησιμοποιεί **απλοποιημένες** γραμμές ισολογισμού· το **Β**
(πλήρες ΕΛΠ) έχει **αναλυτική** δομή. Π.χ. ο `11800` υπάρχει μόνο στο Γ (=Απαιτήσεις)· στο Β **δεν υπάρχει**.

## Ανάλυση ανά πεδίο

### 1. `totalActive` — Σύνολο Ενεργητικού
- **Γ:** `10000` (φύλλο **Γ.1**) — «Σύνολο ενεργητικού».
- **Β:** `10000` (φύλλα **Β.1.1 / Β.1.2** ατομικός, **Β.7.1 / Β.7.2** ενοποιημένος) — «Σύνολο ενεργητικού».
- Ταυτότητα: `11000 + 12000 = 10000` (Β), `10000 = 30000`.

### 2. `businessCircle` — Καθαρό Ύψος Κύκλου Εργασιών
- **Γ:** `40000` (φύλλο **Γ.2**) — «Κύκλος εργασιών (καθαρός)».
- **Β:** `40000` (φύλλα **Β.2.1 / Β.2.2** ατομική κατάσταση αποτελεσμάτων, **Β.8.1 / Β.8.2** ενοποιημένη) — «Κύκλος εργασιών (καθαρός)».

### 3. `employees` — Μέσος Όρος Απασχολούμενων
- **Γ:** `00034` (φύλλο **Γ** header) — «Μέσος όρος εργαζομένων στη διάρκεια της τρέχουσας περιόδου» (`avg_employees`).
- **Β:** `00034` (φύλλο **Β** header) — ίδιο πεδίο (ατομικά στοιχεία). Επιπλέον για όμιλο: **`000341`** «Μέσος όρος εργαζομένων ομίλου» (`avg_employees_group`, μόνο Β).

### 4. `assets` — Πάγια
- **Γ:** `11000` (φύλλο **Γ.1**) — «Πάγια» (μεικτό· συνοδεύεται από `11600` Αποσβέσεις, `11700` Απομειώσεις).
- **Β:** `11000` (φύλλα **Β.1.1 / Β.1.2**, **Β.7.1 / Β.7.2**) — «**Σύνολο μη κυκλοφορούντων**» (πάγια/μη κυκλοφορούντα ενεργητικά).
- ⚠️ Διαφορά εννοιολογίας: στο Γ ο `11000` είναι απλή γραμμή «Πάγια»· στο Β είναι δομικό υποσύνολο
  (`11100 + 11200 + 11300 + 11400 + 11500 = 11000`).

### 5. `requirements` — Απαιτήσεις
- **Γ:** `11800` (φύλλο **Γ.1**) — «Απαιτήσεις» (ενιαία απλοποιημένη γραμμή ΠΜΕ).
- **Β:** ⚠️ **Δεν υπάρχει ενιαίος κωδικός «Απαιτήσεις».** Η αναλυτική δομή του Β (κάτω από `12200`
  «Σύνολο χρηματοοικονομικών στοιχείων και προκαταβολών») περιλαμβάνει:
  - `12210` — Εμπορικές απαιτήσεις
  - `12230` — Λοιπές απαιτήσεις
  - (`11410` — Δάνεια και απαιτήσεις, στα μη κυκλοφορούντα)
  - Φύλλα: **Β.1.1 / Β.1.2**, **Β.7.1 / Β.7.2**.

### 6. `prepayments` — Προκαταβολές και Έσοδα Εισπρακτέα
- **Γ:** `12300` (φύλλο **Γ.1**) — «Προκαταβολές και έσοδα εισπρακτέα» (ενιαία γραμμή ΠΜΕ).
- **Β:** ⚠️ **Δεν υπάρχει ενιαίος κωδικός.** Τα αντίστοιχα αναλυτικά πεδία (κάτω από `12200`):
  - `12250` — Προπληρωμένα έξοδα
  - `12220` — Δουλευμένα έσοδα περιόδου
  - (`12150` — Προκαταβολές για αποθέματα, στα αποθέματα)
  - Φύλλα: **Β.1.1 / Β.1.2**, **Β.7.1 / Β.7.2**.

### 7. `reserves` — Αποθέματα
- **Γ:** `12100` (φύλλο **Γ.1**) — «Αποθέματα».
- **Β:** `12100` (φύλλα **Β.1.1 / Β.1.2**, **Β.7.1 / Β.7.2**) — «**Σύνολο αποθεμάτων**»
  (`12110 + 12120 + 12130 + 12140 + 12150 + 12160 = 12100`).

### 8. `equityTotal` — Σύνολο Καθαρής Θέσης
- **Γ:** `20000` (φύλλο **Γ.1**) — «**Κεφάλαια και αποθεματικά**» (= καθαρή θέση για ΠΜΕ).
- **Β:** `20000` (φύλλα **Β.1.1 / Β.1.2**, **Β.7.1 / Β.7.2**) — «**Σύνολο καθαρής θέσης**».
  - Ενοποιημένα: `21000 + 22000 = 20000` (ιδιοκτήτες μητρικής + δικαιώματα μη ελέγχου).

### 9. `obligationsTotal` — Σύνολο Υποχρεώσεων
- **Γ:** ⚠️ **Δεν υπάρχει ενιαίος κωδικός «Σύνολο Υποχρεώσεων».** Στο απλοποιημένο Γ.1 υπάρχουν:
  - `31100` — Μακροπρόθεσμες υποχρεώσεις
  - `31200` — Βραχυπρόθεσμες υποχρεώσεις
  - Σύνολο υποχρεώσεων = `31100 + 31200` (= `30000 − 20000`).
- **Β:** `31000` (φύλλα **Β.1.1 / Β.1.2**, **Β.7.1 / Β.7.2**) — «**Σύνολο υποχρεώσεων**»
  (`31100 + 31200 = 31000`). Σημ.: στο Β υπάρχει και `32000` (Σύνολο προβλέψεων), ξεχωριστά από τις υποχρεώσεις.

### 10. `resultsNoTax` — Αποτελέσματα προ Φόρων
- **Γ:** `61000` (φύλλο **Γ.2**) — «Αποτέλεσμα προ φόρων».
- **Β:** `61000` (φύλλα **Β.2.1 / Β.2.2**, **Β.8.1 / Β.8.2**) — «Αποτέλεσμα προ φόρων».
- Ταυτότητα: `62000 + 45000 + 55000 = 61000`· `61000 + 56000 = 60000`.

## Σημειώσεις υλοποίησης

- Στο **Γ** όλοι οι κωδικοί επικυρώνονται σε **ένα** μοντέλο (`CompanyHeaderModel`, flat mode).
- Στο **Β** κάθε φύλλο έχει δικό του μοντέλο (per-sheet)· οι κωδικοί ισολογισμού (`10000`, `11000`,
  `12100`, `20000`, `31000`) εμφανίζονται **τόσο** στον ατομικό (Β.1.x) **όσο** και στον ενοποιημένο (Β.7.x)·
  οι κωδικοί αποτελεσμάτων (`40000`, `61000`) σε Β.2.x και Β.8.x.
- Τρία από τα 10 πεδία **δεν** έχουν 1:1 αντιστοιχία σε αμφότερα τα υποδείγματα:
  - `requirements` & `prepayments` → ενιαία γραμμή στο **Γ**, αναλυτικά πεδία στο **Β**.
  - `obligationsTotal` → ενιαίος κωδικός (`31000`) στο **Β**, άθροισμα (`31100+31200`) στο **Γ**.
